The computational model is based on a stack machine in that these instructions manipulate values on an implicit operand stack. In addition to dynamic operands from the stack, some instructions also have static immediate parameters which are part of the instruction itself, typically indices or type annotations. Some instructions are structured in that they bracket nested sequences of instructions.

The following sections categorize instructions into a number of different groups.

Structured Control¶

The \(\mathsf{block}\), \(\mathsf{loop}\) and \(\mathsf{if}\) instructions are structured control instructions. They bracket nested sequences of instructions, called blocks, terminated with \(\mathsf{end}\) or separated by \(\mathsf{else}\) markers. As the grammar prescribes, these instructures must be well-nested.

Each structured control instruction introduces an implicit label. Labels are targets for branch instructions \(\mathsf{br}\) that are referenced by label indices. Unlike with other index spaces, indexing of labels is relative by nesting depth, that is, label \(0\) refers to the innermost structured control instruction enclosing the referring branch instruction, while increasing indices refer to those furhter out. Consequently, labels can only be referenced from within the associated structured control instruction.

This also implies that branches can only be directed outwards, “breaking” from the block of the control construct they target. The exact effect depends on that control construct. In case of \(\mathsf{block}\) or \(\mathsf{if}\) it is a forward jump, resuming execution after the matching \(\mathsf{end}\). In case of \(\mathsf{loop}\) it is a backward jump to the beginning of the loop.

Note

This enforces structured control flow. Intuitively, a branch targeting a \(\mathsf{block}\) or \(\mathsf{if}\) behaves like a \(\mathsf{break}\) statement, while a branch targeting a \(\mathsf{loop}\) behaves like a \(\mathsf{continue}\) statement.

Memory Instructions¶

Linear memory is accessed by \(\mathsf{load}\) and \(\mathsf{store}\) instructions for the different value types. They all take a memory immediate \(\href{../syntax/instructions.html#syntax-memop}{\mathit{memop}}\) that contains a static address offset and an alignment hint.

Integer loads and stores can optionally specify a storage size that is smaller than the width of the respective value type. In the case of loads, a sign extension mode \(\href{../syntax/instructions.html#syntax-sx}{\mathit{sx}}\) is required to select appropriate behavior.
