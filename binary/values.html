Values¶

Bytes¶

Bytes encode themselves.

\[\begin{split}\begin{array}{llcll@{\qquad}l} \def\mathdef763#1{{}}\mathdef763{byte} & \href{../binary/values.html#binary-byte}{\mathtt{byte}} &::=& \def\mathdef817#1{\mathtt{0x#1}}\mathdef817{00} &\Rightarrow& \def\mathdef818#1{\mathtt{0x#1}}\mathdef818{00} \\ &&|&& \dots \\ &&|& \def\mathdef819#1{\mathtt{0x#1}}\mathdef819{FF} &\Rightarrow& \def\mathdef820#1{\mathtt{0x#1}}\mathdef820{FF} \\ \end{array}\end{split}\]

Integers¶

All integers are encoded using the LEB128 variable-length integer encoding, in either unsigned or signed variant.

Unsigned integers are encoded in unsigned LEB128 format. As an additional constraint, the total number of bytes encoding a value of type \(\href{../syntax/values.html#syntax-int}{\def\mathdef767#1{{\mathit{u}#1}}\mathdef767{N}}\) must not exceed \(\mathrm{ceil}(N/7)\) bytes.

\[\begin{split}\begin{array}{llclll@{\qquad}l} \def\mathdef763#1{{}}\mathdef763{unsigned integer} & \href{../binary/values.html#binary-int}{\def\mathdef786#1{{\mathtt{u}#1}}\mathdef786{N}} &::=& n{:}\href{../binary/values.html#binary-byte}{\mathtt{byte}} &\Rightarrow& n & (n < 2^7 \wedge n < 2^N) \\ &&|& n{:}\href{../binary/values.html#binary-byte}{\mathtt{byte}}~~m{:}\def\mathdef821#1{{\mathtt{u}#1}}\mathdef821{(N\mathtt{-7})} &\Rightarrow& 2^7\cdot m + (n-2^7) & (n \geq 2^7 \wedge N > 7) \\ \end{array}\end{split}\]

Signed integers are encoded in signed LEB128 format, which uses a 2’s complement representation. As an additional constraint, the total number of bytes encoding a value of type \(\href{../syntax/values.html#syntax-int}{\def\mathdef774#1{{\mathit{s}#1}}\mathdef774{N}}\) must not exceed \(\mathrm{ceil}(N/7)\) bytes.

\[\begin{split}\begin{array}{llclll@{\qquad}l} \def\mathdef763#1{{}}\mathdef763{signed integer} & \href{../binary/values.html#binary-int}{\def\mathdef792#1{{\mathtt{s}#1}}\mathdef792{N}} &::=& n{:}\href{../binary/values.html#binary-byte}{\mathtt{byte}} &\Rightarrow& n & (n < 2^6 \wedge n < 2^{N-1}) \\ &&|& n{:}\href{../binary/values.html#binary-byte}{\mathtt{byte}} &\Rightarrow& n-2^7 & (2^6 \leq n < 2^7 \wedge n \geq 2^7-2^{N-1}) \\ &&|& n{:}\href{../binary/values.html#binary-byte}{\mathtt{byte}}~~m{:}\def\mathdef822#1{{\mathtt{s}#1}}\mathdef822{(N\mathtt{-7})} &\Rightarrow& 2^7\cdot m + (n-2^7) & (n \geq 2^7 \wedge N > 7) \\ \end{array}\end{split}\]

Uninterpreted integers are encoded as signed integers.

\[\begin{array}{llclll@{\qquad\qquad}l} \def\mathdef763#1{{}}\mathdef763{uninterpreted integer} & \href{../binary/values.html#binary-int}{\def\mathdef795#1{{\mathtt{i}#1}}\mathdef795{N}} &::=& n{:}\href{../binary/values.html#binary-int}{\def\mathdef792#1{{\mathtt{s}#1}}\mathdef792{N}} &\Rightarrow& i & (n = \href{../execution/numerics.html#aux-signed}{\mathrm{signed}}_{\href{../syntax/values.html#syntax-int}{\def\mathdef779#1{{\mathit{i}#1}}\mathdef779{N}}}(i)) \end{array}\]

Note

While the side conditions \(N > 7\) in the productions for non-terminating bytes restrict the length of the \(\def\mathdef823#1{{\mathit{u}#1}}\mathdef823{}\) and \(\def\mathdef824#1{{\mathit{s}#1}}\mathdef824{}\) encodings, “trailing zeros” are still allowed within these bounds. For example, \(\def\mathdef825#1{\mathtt{0x#1}}\mathdef825{03}\) and \(\def\mathdef826#1{\mathtt{0x#1}}\mathdef826{83}~\def\mathdef827#1{\mathtt{0x#1}}\mathdef827{00}\) are both well-formed encodings for the value \(3\) as a \(\href{../syntax/values.html#syntax-int}{\def\mathdef770#1{{\mathit{u}#1}}\mathdef770{\mathit{8}}}\). Similarly, either of \(\def\mathdef828#1{\mathtt{0x#1}}\mathdef828{7e}\) and \(\def\mathdef829#1{\mathtt{0x#1}}\mathdef829{FE}~\def\mathdef830#1{\mathtt{0x#1}}\mathdef830{7F}\) and \(\def\mathdef831#1{\mathtt{0x#1}}\mathdef831{FE}~\def\mathdef832#1{\mathtt{0x#1}}\mathdef832{FF}~\def\mathdef833#1{\mathtt{0x#1}}\mathdef833{7F}\) are well-formed encodings of the value \(-2\) as a \(\href{../syntax/values.html#syntax-int}{\def\mathdef776#1{{\mathit{s}#1}}\mathdef776{\mathit{16}}}\).

The side conditions on the value \(n\) of terminating bytes further enforce that any unused bits in these bytes must be \(0\) for positive values and \(1\) for negative ones. For example, \(\def\mathdef834#1{\mathtt{0x#1}}\mathdef834{83}~\def\mathdef835#1{\mathtt{0x#1}}\mathdef835{10}\) is malformed as a \(\href{../syntax/values.html#syntax-int}{\def\mathdef770#1{{\mathit{u}#1}}\mathdef770{\mathit{8}}}\) encoding. Similarly, both \(\def\mathdef836#1{\mathtt{0x#1}}\mathdef836{83}~\def\mathdef837#1{\mathtt{0x#1}}\mathdef837{3E}\) and \(\def\mathdef838#1{\mathtt{0x#1}}\mathdef838{FF}~\def\mathdef839#1{\mathtt{0x#1}}\mathdef839{7B}\) are malformed as \(\href{../syntax/values.html#syntax-int}{\def\mathdef775#1{{\mathit{s}#1}}\mathdef775{\mathit{8}}}\) encodings.

Floating-Point¶

Floating point values are encoded by their IEEE 754 bit pattern in little endian byte order:

\[\begin{split}\begin{array}{llclll@{\qquad\qquad}l} \def\mathdef763#1{{}}\mathdef763{floating-point number} & \href{../binary/values.html#binary-float}{\def\mathdef798#1{{\mathtt{f}#1}}\mathdef798{N}} &::=& b^\ast{:\,}\href{../binary/values.html#binary-byte}{\mathtt{byte}}^{N/8} &\Rightarrow& \href{../execution/numerics.html#aux-bytes}{\mathrm{bytes}}_{\href{../syntax/values.html#syntax-float}{\def\mathdef782#1{{\mathit{f}#1}}\mathdef782{N}}}^{-1}(\mathrm{reverse}(b^\ast)) \\ \end{array}\end{split}\]

Here, \(\mathrm{reverse}(b^\ast)\) denotes the byte sequence \(b^\ast\) in reversed order.

Vectors¶

Vectors are encoded with their length followed by the encoding of their element sequence.

\[\begin{split}\begin{array}{llclll@{\qquad\qquad}l} \def\mathdef763#1{{}}\mathdef763{vector} & \href{../binary/values.html#binary-vec}{\mathtt{vec}}(\mathtt{B}) &::=& n{:}\href{../binary/values.html#binary-int}{\def\mathdef790#1{{\mathtt{u}#1}}\mathdef790{\mathtt{32}}}~~(x{:}\mathtt{B})^n &\Rightarrow& x^n \\ \end{array}\end{split}\]

Names¶

Names are encoded like a vector of bytes containing the Unicode UTF-8 encoding of the name’s code point sequence.

\[\begin{split}\begin{array}{llclll@{\qquad}l} \def\mathdef763#1{{}}\mathdef763{name} & \href{../binary/values.html#binary-name}{\mathtt{name}} &::=& n{:}\href{../binary/values.html#binary-int}{\def\mathdef790#1{{\mathtt{u}#1}}\mathdef790{\mathtt{32}}}~~(\mathit{uc}{:}\href{../binary/values.html#binary-name}{\mathtt{codepoint}})^\ast &\Rightarrow& \mathit{uc}^\ast & (|\href{../binary/values.html#binary-name}{\mathtt{codepoint}}^\ast| = n) \\ \def\mathdef763#1{{}}\mathdef763{code point} & \href{../binary/values.html#binary-name}{\mathtt{codepoint}} &::=& \mathit{uv}{:}\href{../binary/values.html#binary-name}{\mathtt{codeval}}_N &\Rightarrow& \mathit{uv} & (\mathit{uv} \geq N \wedge (\mathit{uv} < \def\mathdef840#1{\mathrm{U{+}#1}}\mathdef840{D800} \vee \def\mathdef841#1{\mathrm{U{+}#1}}\mathdef841{E000} \leq \mathit{uv} < \def\mathdef842#1{\mathrm{U{+}#1}}\mathdef842{110000})) \\ \def\mathdef763#1{{}}\mathdef763{code value} & \href{../binary/values.html#binary-name}{\mathtt{codeval}}_N &::=& b_1{:}\href{../binary/values.html#binary-byte}{\mathtt{byte}} &\Rightarrow& b_1 & (b_1 < \def\mathdef843#1{\mathtt{0x#1}}\mathdef843{80} \wedge N = \def\mathdef844#1{\mathrm{U{+}#1}}\mathdef844{00}) \\ &&|& b_1{:}\href{../binary/values.html#binary-byte}{\mathtt{byte}}~~b_2{:}\href{../binary/values.html#binary-name}{\mathtt{cont}} &\Rightarrow& 2^6\cdot(b_1-\def\mathdef845#1{\mathtt{0x#1}}\mathdef845{c0}) + b_2 & (\def\mathdef846#1{\mathtt{0x#1}}\mathdef846{c0} \leq b_1 < \def\mathdef847#1{\mathtt{0x#1}}\mathdef847{e0} \wedge N = \def\mathdef848#1{\mathrm{U{+}#1}}\mathdef848{80}) \\ &&|& b_1{:}\href{../binary/values.html#binary-byte}{\mathtt{byte}}~~b_2{:}\href{../binary/values.html#binary-name}{\mathtt{cont}}~~b_3{:}\href{../binary/values.html#binary-name}{\mathtt{cont}} &\Rightarrow& 2^{12}\cdot(b_1-\def\mathdef849#1{\mathtt{0x#1}}\mathdef849{e0}) + 2^6\cdot b_2 + b_3 & (\def\mathdef850#1{\mathtt{0x#1}}\mathdef850{e0} \leq b_1 < \def\mathdef851#1{\mathtt{0x#1}}\mathdef851{f0} \wedge N = \def\mathdef852#1{\mathrm{U{+}#1}}\mathdef852{800}) \\ &&|& b_1{:}\href{../binary/values.html#binary-byte}{\mathtt{byte}}~~b_2{:}\href{../binary/values.html#binary-name}{\mathtt{cont}}~~b_3{:}\href{../binary/values.html#binary-name}{\mathtt{cont}}~~b_4{:}\href{../binary/values.html#binary-name}{\mathtt{cont}} &\Rightarrow& 2^{18}\cdot(b_1-\def\mathdef853#1{\mathtt{0x#1}}\mathdef853{f0}) + 2^{12}\cdot b_2 + 2^6\cdot b_3 + b_4 & (\def\mathdef854#1{\mathtt{0x#1}}\mathdef854{f0} \leq b_1 < \def\mathdef855#1{\mathtt{0x#1}}\mathdef855{f8} \wedge N = \def\mathdef856#1{\mathrm{U{+}#1}}\mathdef856{10000}) \\ \def\mathdef763#1{{}}\mathdef763{code continuation} & \href{../binary/values.html#binary-name}{\mathtt{cont}} &::=& b{:}\href{../binary/values.html#binary-byte}{\mathtt{byte}} &\Rightarrow& b - \def\mathdef857#1{\mathtt{0x#1}}\mathdef857{80} & (b \geq \def\mathdef858#1{\mathtt{0x#1}}\mathdef858{80}) \\ \end{array}\end{split}\]

Note

The size, \(||\href{../binary/values.html#binary-name}{\mathtt{codepoint}}^\ast||\) denotes the number of bytes in the encoding of the sequence, not the number of code points.

The index \(N\) to \(\href{../binary/values.html#binary-name}{\mathtt{codeval}}\) is the minimum value that a given byte sequence may decode into. The respective side conditions on it exclude encodings using more than the minimal number of bytes to represent a code point.
