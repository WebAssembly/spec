Modules¶

Modules are valid when all the definitions they contain are valid. To that end, each definition is classified with a suitable type.

Auxiliary Rules¶

Limits \(\{ \mathsf{min}~n, \mathsf{max}~m^? \}\)¶

  • If the maximum \(m^?\) is not empty, then its value must not be smaller than \(n\).
  • Then the limit is valid.
\[\frac{ (n \leq m)^? }{ \vdash \{ \mathsf{min}~n, \mathsf{max}~m^? \} ~\mathrm{ok} }\]

Functions¶

Functions \(\href{../syntax/modules.html#syntax-func}{\mathit{func}}\) are classified by function types of the form \([t_1^\ast] \to [t_2^?]\).

\(\{ \mathsf{type}~x, \mathsf{locals}~t^\ast, \mathsf{body}~\href{../syntax/instructions.html#syntax-expr}{\mathit{expr}} \}\)¶

  • The type \(C.\mathsf{types}[x]\) must be defined in the context.
  • Let \([t_1^\ast] \to [t_2^\ast]\) be the function type \(C.\mathsf{types}[x]\).
  • The length of \(t_2^\ast\) must not be larger than \(1\).
  • Let \(C'\) be the same context as \(C\), but with:
    • \(\mathsf{locals}\) set to the sequence of value types \(t_1^\ast~t^\ast\), concatenating parameters and locals,
    • \(\mathsf{labels}\) set to the singular sequence containing only result type \([t_2^\ast]\).
    • \(\mathsf{return}\) set to the result type \([t_2^\ast]\).
  • Under the context \(C'\), the expression \(\href{../syntax/instructions.html#syntax-expr}{\mathit{expr}}\) must be valid with type \(t_2^\ast\).
  • Then the function definition is valid with type \([t_1^\ast] \to [t_2^\ast]\).
\[\frac{ C.\mathsf{types}[x] = [t_1^\ast] \to [t_2^?] \qquad C,\mathsf{locals}\,t_1^\ast~t^\ast,\mathsf{return}~[t_2^?] \vdash \href{../syntax/instructions.html#syntax-expr}{\mathit{expr}} : [t_2^?] }{ C \vdash \{ \mathsf{type}~x, \mathsf{locals}~t^\ast, \mathsf{body}~\href{../syntax/instructions.html#syntax-expr}{\mathit{expr}} \} : [t_1^\ast] \to [t_2^?] }\]

Note

The restriction on the length of the result types \(t_2^\ast\) may be lifted in future versions of WebAssembly.

Tables¶

Tables \(\href{../syntax/modules.html#syntax-table}{\mathit{table}}\) are classified by table types of the form \(\href{../syntax/types.html#syntax-limits}{\mathit{limits}}~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}}\).

\(\{ \mathsf{type}~\href{../syntax/types.html#syntax-limits}{\mathit{limits}}~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}} \}\)¶

  • The limits \(\href{../syntax/types.html#syntax-limits}{\mathit{limits}}\) must be valid.
  • Then the table definition is valid with type \(\href{../syntax/types.html#syntax-limits}{\mathit{limits}}~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}}\).
\[\frac{ \vdash \href{../syntax/types.html#syntax-limits}{\mathit{limits}}~\mathrm{ok} }{ C \vdash \{ \mathsf{type}~\href{../syntax/types.html#syntax-limits}{\mathit{limits}}~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}} \} : \href{../syntax/types.html#syntax-limits}{\mathit{limits}}~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}} }\]

Memories¶

Memories \(\href{../syntax/modules.html#syntax-mem}{\mathit{mem}}\) are classified by memory types of the form \(\href{../syntax/types.html#syntax-limits}{\mathit{limits}}\).

\(\{ \mathsf{type}~\href{../syntax/types.html#syntax-limits}{\mathit{limits}} \}\)¶

  • The limits \(\href{../syntax/types.html#syntax-limits}{\mathit{limits}}\) must be valid.
  • Then the memory definition is valid with type \(\href{../syntax/types.html#syntax-limits}{\mathit{limits}}\).
\[\frac{ \vdash \href{../syntax/types.html#syntax-limits}{\mathit{limits}}~\mathrm{ok} }{ C \vdash \{ \mathsf{type}~\href{../syntax/types.html#syntax-limits}{\mathit{limits}} \} : \href{../syntax/types.html#syntax-limits}{\mathit{limits}}~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}} }\]

Globals¶

Globals \(\href{../syntax/modules.html#syntax-global}{\mathit{global}}\) are classified by global types of the form \(\href{../syntax/types.html#syntax-mut}{\mathit{mut}}~t\).

\(\{ \mathsf{type}~\href{../syntax/types.html#syntax-mut}{\mathit{mut}}~t, \mathsf{init}~\href{../syntax/instructions.html#syntax-expr}{\mathit{expr}} \}\)¶

  • The expression \(\href{../syntax/instructions.html#syntax-expr}{\mathit{expr}}\) must be valid with result type \([t]\).
  • The expression \(\href{../syntax/instructions.html#syntax-expr}{\mathit{expr}}\) must be constant.
  • Then the global definition is valid with type \(\href{../syntax/types.html#syntax-mut}{\mathit{mut}}~t\).
\[\frac{ C \vdash \href{../syntax/instructions.html#syntax-expr}{\mathit{expr}} : [t] \qquad C \vdash \href{../syntax/instructions.html#syntax-expr}{\mathit{expr}} ~\mathrm{const} }{ C \vdash \{ \mathsf{type}~\href{../syntax/types.html#syntax-mut}{\mathit{mut}}~t, \mathsf{init}~\href{../syntax/instructions.html#syntax-expr}{\mathit{expr}} \} : \href{../syntax/types.html#syntax-mut}{\mathit{mut}}~t }\]

Element Segments¶

Element segments \(\href{../syntax/modules.html#syntax-elem}{\mathit{elem}}\) are not classified by a type.

\(\{ \mathsf{table}~x, \mathsf{offset}~\href{../syntax/instructions.html#syntax-expr}{\mathit{expr}}, \mathsf{init}~y^\ast \}\)¶

  • The table \(C.\mathsf{tables}[x]\) must be defined in the context.
  • Let \(\href{../syntax/types.html#syntax-limits}{\mathit{limits}}~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}}\) be the table type \(C.\mathsf{tables}[x]\).
  • The element type \(\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}}\) must be \(\href{../syntax/types.html#syntax-elemtype}{\mathsf{anyfunc}}\).
  • The expression \(\href{../syntax/instructions.html#syntax-expr}{\mathit{expr}}\) must be valid with result type \([\href{../syntax/types.html#syntax-valtype}{\mathsf{i32}}]\).
  • The expression \(\href{../syntax/instructions.html#syntax-expr}{\mathit{expr}}\) must be constant.
  • For each \(y_i\) in \(y^\ast\), the function \(C.\mathsf{funcs}[y]\) must be defined in the context.
  • Then the element segment is valid.
\[\frac{ C.\mathsf{tables}[x] = \href{../syntax/types.html#syntax-limits}{\mathit{limits}}~\href{../syntax/types.html#syntax-elemtype}{\mathsf{anyfunc}} \qquad C \vdash \href{../syntax/instructions.html#syntax-expr}{\mathit{expr}} : [\href{../syntax/types.html#syntax-valtype}{\mathsf{i32}}] \qquad C \vdash \href{../syntax/instructions.html#syntax-expr}{\mathit{expr}} ~\mathrm{const} \qquad (C.\mathsf{funcs}[y] = \href{../syntax/types.html#syntax-functype}{\mathit{functype}})^\ast }{ C \vdash \{ \mathsf{table}~x, \mathsf{offset}~\href{../syntax/instructions.html#syntax-expr}{\mathit{expr}}, \mathsf{init}~y^\ast \} ~\mathrm{ok} }\]

Data Segments¶

Data segments \(\href{../syntax/modules.html#syntax-data}{\mathit{data}}\) are not classified by any type.

\(\{ \mathsf{mem}~x, \mathsf{offset}~\href{../syntax/instructions.html#syntax-expr}{\mathit{expr}}, \mathsf{init}~b^\ast \}\)¶

  • The memory \(C.\mathsf{mems}[x]\) must be defined in the context.
  • The expression \(\href{../syntax/instructions.html#syntax-expr}{\mathit{expr}}\) must be valid with result type \([\href{../syntax/types.html#syntax-valtype}{\mathsf{i32}}]\).
  • The expression \(\href{../syntax/instructions.html#syntax-expr}{\mathit{expr}}\) must be constant.
  • Then the data segment is valid.
\[\frac{ C.\mathsf{mems}[x] = \href{../syntax/types.html#syntax-limits}{\mathit{limits}} \qquad C \vdash \href{../syntax/instructions.html#syntax-expr}{\mathit{expr}} : [\href{../syntax/types.html#syntax-valtype}{\mathsf{i32}}] \qquad C \vdash \href{../syntax/instructions.html#syntax-expr}{\mathit{expr}} ~\mathrm{const} }{ C \vdash \{ \mathsf{mem}~x, \mathsf{offset}~\href{../syntax/instructions.html#syntax-expr}{\mathit{expr}}, \mathsf{init}~b^\ast \} ~\mathrm{ok} }\]

Start Function¶

Start function declarations \(\href{../syntax/modules.html#syntax-start}{\mathit{start}}\) are not classified by any type.

\(\{ \mathsf{func}~x \}\)¶

  • The function \(C.\mathsf{funcs}[x]\) must be defined in the context.
  • The type of \(C.\mathsf{funcs}[x]\) must be \([] \to []\).
  • Then the start function is valid.
\[\frac{ C.\mathsf{funcs}[x] = [] \to [] }{ C \vdash \{ \mathsf{func}~x \} ~\mathrm{ok} }\]

Exports¶

Exports \(\href{../syntax/modules.html#syntax-export}{\mathit{export}}\) are classified by their export name. Export descriptions \(\href{../syntax/modules.html#syntax-exportdesc}{\mathit{exportdesc}}\) are not classified by any type.

\(\{ \mathsf{name}~\href{../syntax/values.html#syntax-name}{\mathit{name}}, \mathsf{desc}~\href{../syntax/modules.html#syntax-exportdesc}{\mathit{exportdesc}} \}\)¶

  • The export description \(\href{../syntax/modules.html#syntax-exportdesc}{\mathit{exportdesc}}\) must be valid with type \(\href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}}\).
  • Then the export is valid with name \(\href{../syntax/values.html#syntax-name}{\mathit{name}}\).
\[\frac{ C \vdash \href{../syntax/modules.html#syntax-exportdesc}{\mathit{exportdesc}} ~\mathrm{ok} }{ C \vdash \{ \mathsf{name}~\href{../syntax/values.html#syntax-name}{\mathit{name}}, \mathsf{desc}~\href{../syntax/modules.html#syntax-exportdesc}{\mathit{exportdesc}} \} : \href{../syntax/values.html#syntax-name}{\mathit{name}} }\]

\(\mathsf{func}~x\)¶

  • The function \(C.\mathsf{funcs}[x]\) must be defined in the context.
  • Then the export description is valid.
\[\frac{ C.\mathsf{funcs}[x] = \href{../syntax/types.html#syntax-functype}{\mathit{functype}} }{ C \vdash \mathsf{func}~x ~\mathrm{ok} }\]

\(\mathsf{table}~x\)¶

  • The table \(C.\mathsf{tables}[x]\) must be defined in the context.
  • Then the export description is valid.
\[\frac{ C.\mathsf{tables}[x] = \href{../syntax/types.html#syntax-tabletype}{\mathit{tabletype}} }{ C \vdash \mathsf{table}~x ~\mathrm{ok} }\]

\(\mathsf{mem}~x\)¶

  • The memory \(C.\mathsf{mems}[x]\) must be defined in the context.
  • Then the export description is valid.
\[\frac{ C.\mathsf{mems}[x] = \href{../syntax/types.html#syntax-memtype}{\mathit{memtype}} }{ C \vdash \mathsf{mem}~x ~\mathrm{ok} }\]

\(\mathsf{global}~x\)¶

  • The global \(C.\mathsf{globals}[x]\) must be defined in the context.
  • Let \(\href{../syntax/types.html#syntax-mut}{\mathit{mut}}~t\) be the global type \(C.\mathsf{globals}[x]\).
  • The mutability \(\href{../syntax/types.html#syntax-mut}{\mathit{mut}}\) must be \(\href{../syntax/types.html#syntax-mut}{\mathsf{var}}\).
  • Then the export description is valid.
\[\frac{ C.\mathsf{globals}[x] = \href{../syntax/types.html#syntax-mut}{\mathsf{var}}~t }{ C \vdash \mathsf{global}~x ~\mathrm{ok} }\]

Imports¶

Imports \(\href{../syntax/modules.html#syntax-import}{\mathit{import}}\) and import descriptions \(\href{../syntax/modules.html#syntax-importdesc}{\mathit{importdesc}}\) are classified by external types.

\(\{ \mathsf{module}~\href{../syntax/values.html#syntax-name}{\mathit{name}}_1, \mathsf{name}~\href{../syntax/values.html#syntax-name}{\mathit{name}}_2, \mathsf{desc}~\href{../syntax/modules.html#syntax-importdesc}{\mathit{importdesc}} \}\)¶

  • The import description \(\href{../syntax/modules.html#syntax-importdesc}{\mathit{importdesc}}\) must be valid with type \(\href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}}\).
  • Then the import is valid with type \(\href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}}\).
\[\frac{ C \vdash \href{../syntax/modules.html#syntax-importdesc}{\mathit{importdesc}} : \href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}} }{ C \vdash \{ \mathsf{module}~\href{../syntax/values.html#syntax-name}{\mathit{name}}_1, \mathsf{name}~\href{../syntax/values.html#syntax-name}{\mathit{name}}_2, \mathsf{desc}~\href{../syntax/modules.html#syntax-importdesc}{\mathit{importdesc}} \} : \href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}} }\]

\(\mathsf{func}~x\)¶

  • The function \(C.\mathsf{types}[x]\) must be defined in the context.
  • Let \([t_1^\ast] \to [t_2^\ast]\) be the function type \(C.\mathsf{types}[x]\).
  • The length of \(t_2^\ast\) must not be larger than \(1\).
  • Then the import description is valid with type \(\mathsf{func}~[t_1^\ast] \to [t_2^\ast]\).
\[\frac{ C.\mathsf{types}[x] = [t_1^\ast] \to [t_2^?] }{ C \vdash \mathsf{func}~x : \mathsf{func}~[t_1^\ast] \to [t_2^?] }\]

Note

The restriction on the length of the result types \(t_2^\ast\) may be lifted in future versions of WebAssembly.

\(\mathsf{table}~\href{../syntax/types.html#syntax-limits}{\mathit{limits}}~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}}\)¶

  • The limits \(\href{../syntax/types.html#syntax-limits}{\mathit{limits}}\) must be valid.
  • Then the import description is valid with type \(\mathsf{table}~\href{../syntax/types.html#syntax-limits}{\mathit{limits}}~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}}\).
\[\frac{ \vdash \href{../syntax/types.html#syntax-limits}{\mathit{limits}} ~\mathrm{ok} }{ C \vdash \mathsf{table}~\href{../syntax/types.html#syntax-limits}{\mathit{limits}}~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}} : \mathsf{table}~\href{../syntax/types.html#syntax-limits}{\mathit{limits}}~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}} }\]

\(\mathsf{mem}~\href{../syntax/types.html#syntax-limits}{\mathit{limits}}\)¶

  • The limits \(\href{../syntax/types.html#syntax-limits}{\mathit{limits}}\) must be valid.
  • Then the import description is valid with type \(\mathsf{mem}~\href{../syntax/types.html#syntax-limits}{\mathit{limits}}\).
\[\frac{ \vdash \href{../syntax/types.html#syntax-limits}{\mathit{limits}} ~\mathrm{ok} }{ C \vdash \mathsf{mem}~\href{../syntax/types.html#syntax-limits}{\mathit{limits}} : \mathsf{mem}~\href{../syntax/types.html#syntax-limits}{\mathit{limits}} }\]

\(\mathsf{global}~\href{../syntax/types.html#syntax-mut}{\mathit{mut}}~t\)¶

  • The mutability \(\href{../syntax/types.html#syntax-mut}{\mathit{mut}}\) must be \(\href{../syntax/types.html#syntax-mut}{\mathsf{var}}\).
  • Then the import description is valid with type \(\mathsf{global}~t\).
\[\frac{ }{ C \vdash \mathsf{global}~\href{../syntax/types.html#syntax-mut}{\mathsf{var}}~t : \mathsf{global}~\href{../syntax/types.html#syntax-mut}{\mathsf{var}}~t }\]

Modules¶

A module is entirely closed, that is, it only refers to definitions that appear in the module itself. Consequently, no initial context is required. Instead, the context \(C\) for validation of the module’s content is constructed from the types of definitions in the module itself.

  • Let \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}\) be the module to validate.
  • Let \(C\) be a context where:
    • \(C.\mathsf{types}\) is \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{types}\),
    • \(C.\mathsf{funcs}\) is \(\href{../execution/runtime.html#syntax-externtype}{\mathrm{funcs}}(\href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}}_i^\ast)\) concatenated with \(\href{../syntax/types.html#syntax-functype}{\mathit{functype}}_i^\ast\), with the type sequences \(\href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}}_i^\ast\) and \(\href{../syntax/types.html#syntax-functype}{\mathit{functype}}_i^\ast\) as determined below,
    • \(C.\mathsf{tables}\) is \(\href{../execution/runtime.html#syntax-externtype}{\mathrm{tables}}(\href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}}_i^\ast)\) concatenated with \(\href{../syntax/types.html#syntax-tabletype}{\mathit{tabletype}}_i^\ast\), with the type sequences \(\href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}}_i^\ast\) and \(\href{../syntax/types.html#syntax-tabletype}{\mathit{tabletype}}_i^\ast\) as determined below,
    • \(C.\mathsf{mems}\) is \(\href{../execution/runtime.html#syntax-externtype}{\mathrm{mems}}(\href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}}_i^\ast)\) concatenated with \(\href{../syntax/types.html#syntax-memtype}{\mathit{memtype}}_i^\ast\), with the type sequences \(\href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}}_i^\ast\) and \(\href{../syntax/types.html#syntax-memtype}{\mathit{memtype}}_i^\ast\) as determined below,
    • \(C.\mathsf{globals}\) is \(\href{../execution/runtime.html#syntax-externtype}{\mathrm{globals}}(\href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}}_i^\ast)\) concatenated with \(\href{../syntax/types.html#syntax-globaltype}{\mathit{globaltype}}_i^\ast\), with the type sequences \(\href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}}_i^\ast\) and \(\href{../syntax/types.html#syntax-globaltype}{\mathit{globaltype}}_i^\ast\) as determined below.
    • \(C.\mathsf{locals}\) is empty,
    • \(C.\mathsf{labels}\) is empty.
    • \(C.\mathsf{return}\) is empty.
  • Under the context \(C\):
    • For each \(\href{../syntax/modules.html#syntax-func}{\mathit{func}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{funcs}\), the definition \(\href{../syntax/modules.html#syntax-func}{\mathit{func}}_i\) must be valid with a function type \(\href{../syntax/types.html#syntax-functype}{\mathit{functype}}_i\).
    • For each \(\href{../syntax/modules.html#syntax-table}{\mathit{table}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{tables}\), the definition \(\href{../syntax/modules.html#syntax-table}{\mathit{table}}_i\) must be valid with a table type \(\href{../syntax/types.html#syntax-tabletype}{\mathit{tabletype}}_i\).
    • For each \(\href{../syntax/modules.html#syntax-mem}{\mathit{mem}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{mems}\), the definition \(\href{../syntax/modules.html#syntax-mem}{\mathit{mem}}_i\) must be valid with a memory type \(\href{../syntax/types.html#syntax-memtype}{\mathit{memtype}}_i\).
    • For each \(\href{../syntax/modules.html#syntax-global}{\mathit{global}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{globals}\):
      • Let \(C_i\) be the context where \(C_i.\mathsf{globals}\) is the sequence \(\href{../execution/runtime.html#syntax-externtype}{\mathrm{globals}}(\href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}}_i^\ast)\) concatenated with \(\href{../syntax/types.html#syntax-globaltype}{\mathit{globaltype}}_0~\dots~\href{../syntax/types.html#syntax-globaltype}{\mathit{globaltype}}_{i-1}\), and all other fields are empty.
      • Under the context \(C_i\), the definition \(\href{../syntax/modules.html#syntax-global}{\mathit{global}}_i\) must be valid with a global type \(\href{../syntax/types.html#syntax-globaltype}{\mathit{globaltype}}_i\).
    • For each \(\href{../syntax/modules.html#syntax-elem}{\mathit{elem}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{elem}\), the segment \(\href{../syntax/modules.html#syntax-elem}{\mathit{elem}}_i\) must be valid.
    • For each \(\href{../syntax/modules.html#syntax-data}{\mathit{data}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{data}\), the segment \(\href{../syntax/modules.html#syntax-elem}{\mathit{elem}}_i\) must be valid.
    • If \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{start}\) is non-empty, then \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{start}\) must be valid.
    • For each \(\href{../syntax/modules.html#syntax-import}{\mathit{import}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{imports}\), the segment \(\href{../syntax/modules.html#syntax-import}{\mathit{import}}_i\) must be valid with an external type \(\href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}}_i\).
    • For each \(\href{../syntax/modules.html#syntax-export}{\mathit{export}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{exports}\), the segment \(\href{../syntax/modules.html#syntax-import}{\mathit{import}}_i\) must be valid with a name \(\href{../syntax/values.html#syntax-name}{\mathit{name}}_i\).
  • The length of \(C.\mathsf{tables}\) must not be larger than \(1\).
  • The length of \(C.\mathsf{mems}\) must not be larger than \(1\).
  • All export names \(\href{../syntax/values.html#syntax-name}{\mathit{name}}_i\) must be different.
  • Let \(\href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}}^\ast\) be the concatenation of external types \(\href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}}_i\) of the imports, in index order.
  • Then the module is valid with external types \(\href{../execution/runtime.html#syntax-externtype}{\mathit{externtype}}^\ast\).
\[\begin{split}\frac{ \begin{array}{@{}c@{}} (C \vdash \href{../syntax/modules.html#syntax-func}{\mathit{func}} : \mathit{ft})^\ast \quad (C \vdash \href{../syntax/modules.html#syntax-table}{\mathit{table}} : \mathit{tt})^\ast \quad (C \vdash \href{../syntax/modules.html#syntax-mem}{\mathit{mem}} : \mathit{mt})^\ast \quad (C' \vdash \href{../syntax/modules.html#syntax-global}{\mathit{global}} : \mathit{gt})^\ast \\ (C \vdash \href{../syntax/modules.html#syntax-elem}{\mathit{elem}} ~\mathrm{ok})^\ast \quad (C \vdash \href{../syntax/modules.html#syntax-data}{\mathit{data}} ~\mathrm{ok})^\ast \quad (C \vdash \href{../syntax/modules.html#syntax-start}{\mathit{start}} ~\mathrm{ok})^? \quad (C \vdash \href{../syntax/modules.html#syntax-import}{\mathit{import}} : \mathit{it})^\ast \quad (C \vdash \href{../syntax/modules.html#syntax-export}{\mathit{export}} : \mathit{name})^\ast \\ \mathit{ift}^\ast = \href{../execution/runtime.html#syntax-externtype}{\mathrm{funcs}}(\mathit{it}^\ast) \qquad \mathit{itt}^\ast = \href{../execution/runtime.html#syntax-externtype}{\mathrm{tables}}(\mathit{it}^\ast) \qquad \mathit{imt}^\ast = \href{../execution/runtime.html#syntax-externtype}{\mathrm{mems}}(\mathit{it}^\ast) \qquad \mathit{igt}^\ast = \href{../execution/runtime.html#syntax-externtype}{\mathrm{globals}}(\mathit{it}^\ast) \\ C = \{ \mathsf{types}~\href{../syntax/types.html#syntax-functype}{\mathit{functype}}^\ast, \mathsf{funcs}~\mathit{ift}^\ast~\mathit{ft}^\ast, \mathsf{tables}~\mathit{itt}^\ast~\mathit{tt}^\ast, \mathsf{mems}~\mathit{imt}^\ast~\mathit{mt}^\ast, \mathsf{globals}~\mathit{igt}^\ast~\mathit{gt}^\ast \} \\ C' = \{ \mathsf{globals}~\mathit{igt}^\ast \} \qquad |C.\mathsf{tables}| \leq 1 \qquad |C.\mathsf{mems}| \leq 1 \qquad \href{../syntax/values.html#syntax-name}{\mathit{name}}^\ast ~\mathrm{disjoint} \end{array} }{ \vdash \{ \begin{array}[t]{@{}l@{}} \mathsf{types}~\href{../syntax/types.html#syntax-functype}{\mathit{functype}}^\ast, \mathsf{funcs}~\href{../syntax/modules.html#syntax-func}{\mathit{func}}^\ast, \mathsf{tables}~\href{../syntax/modules.html#syntax-table}{\mathit{table}}^\ast, \mathsf{mems}~\href{../syntax/modules.html#syntax-mem}{\mathit{mem}}^\ast, \mathsf{globals}~\href{../syntax/modules.html#syntax-global}{\mathit{global}}^\ast, \\ \mathsf{elem}~\href{../syntax/modules.html#syntax-elem}{\mathit{elem}}^\ast, \mathsf{data}~\href{../syntax/modules.html#syntax-data}{\mathit{data}}^\ast, \mathsf{start}~\href{../syntax/modules.html#syntax-start}{\mathit{start}}^?, \mathsf{imports}~\href{../syntax/modules.html#syntax-import}{\mathit{import}}^\ast, \mathsf{exports}~\href{../syntax/modules.html#syntax-export}{\mathit{export}}^\ast \} : \mathit{it}^\ast \\ \end{array} }\end{split}\]

Note

Most definitions in a module – particularly functions – are mutually recursive. Consequently, the definition of the context \(C\) in this rule is recursive: it depends on the outcome of validation of the function, table, memory, and global definitions contained in the module, which itself depends on \(C\). However, this recursion is just a specification device. All types needed to construct \(C\) can easily be determined from a simple pre-pass over the module that does not perform any actual validation.

Globals, however, are not recursive. The effect of defining the limited context \(C'\) for validating the module’s globals is that their initialization expressions can only access imported globals and nothing else.

Note

The restriction on the number of tables and memories may be lifted in future versions of WebAssembly.
